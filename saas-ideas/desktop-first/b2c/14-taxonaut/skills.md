# Taxonaut -- Skills

> Technical, domain, design, and business skills required to build and operate an AI-powered tax strategy platform.

---

## Important Disclaimer

Taxonaut provides **tax education and strategy suggestions** for informational purposes only. It does not provide tax advice, legal advice, or accounting services. All recommendations include disclaimers directing users to consult qualified tax professionals. This distinction is legally critical -- it determines regulatory requirements, liability exposure, and marketing language.

**What Taxonaut IS**: A tax education and strategy suggestion tool
**What Taxonaut IS NOT**: A CPA firm, tax advisor, law firm, or registered investment advisor

Every piece of content generated by the AI must include appropriate disclaimers. The UI must surface these disclaimers without burying them.

## Technical Skills

### 1. Electron Development

Building a desktop application with Electron requires specific expertise beyond standard web development.

**Core Skills:**
- Electron main process vs renderer process architecture
- IPC (Inter-Process Communication) between main and renderer
- Electron security best practices (contextIsolation, nodeIntegration disabled, Content Security Policy)
- Auto-updater implementation (Squirrel for macOS/Windows)
- Code signing and notarization (Apple Developer Program, Windows Authenticode)
- Native OS integration (system tray, notifications, file system, keychain)
- SQLite integration in main process for local data caching
- Electron Forge or electron-builder for packaging and distribution
- Memory management and performance profiling for desktop apps
- Deep linking (`taxonaut://` protocol registration)
- Crash reporting (Sentry integration in Electron)

**Why This Skill Is Unique:**
Most React developers have never built an Electron app. The main process/renderer process split, IPC communication patterns, and native OS integration create a learning curve. Security is also different -- browser security models do not fully apply, and Electron-specific vulnerabilities (like nodeIntegration) must be understood.

**Learning Resources:**
- Electron official documentation: https://www.electronjs.org/docs
- "Electron in Action" by Steve Kinney (Manning Publications)
- Electron Fiddle for rapid prototyping
- electron-builder documentation for packaging
- Apple Developer documentation for notarization

### 3. Plaid API Integration

Connecting to bank accounts is the foundation of Taxonaut.

**Core Skills:**
- Plaid Link SDK integration (web view in Electron)
- Plaid API: Auth, Transactions, Balance, Identity products
- Webhook handling for real-time transaction updates
- Token management (access tokens, item management)
- Error handling (institution-specific errors, credential rotation)
- Plaid Exchange for itemized transactions
- Sandbox vs Development vs Production environments
- Handling Plaid rate limits and pagination
- Institution coverage gaps and fallback strategies
- Plaid's transaction categorization vs custom categorization

**Key Challenges:**
- Plaid tokens expire and require user re-authentication (handle gracefully)
- Some institutions have limited API support (certain credit unions)
- Transaction descriptions are messy and vary wildly by bank
- Historical transaction limits vary by institution (some only 90 days)
- Webhook reliability (need idempotent processing)

**Learning Resources:**
- Plaid documentation: https://plaid.com/docs
- Plaid Quickstart guides (Node.js)
- Plaid community Slack channel

### 4. Financial Data Modeling

Handling money in software is harder than it looks.

**Core Skills:**
- Decimal arithmetic (never use floating point for money -- use Decimal.js or integer cents)
- Double-entry bookkeeping concepts
- Transaction reconciliation
- Currency formatting and localization
- Financial data normalization (Plaid descriptions to clean merchant names)
- Time-series data modeling for income/expense trends
- Tax bracket calculation algorithms (progressive taxation)
- Present value and future value calculations (for retirement planning)
- Amortization calculations (for depreciation)

**Critical Rule:**
Never use JavaScript `Number` type for monetary calculations. Use integer cents (amount * 100) or a decimal library. Floating point rounding errors in financial software are a lawsuit waiting to happen.

```typescript
// WRONG
const tax = income * 0.28; // floating point error

// RIGHT
import Decimal from 'decimal.js';
const tax = new Decimal(income).times('0.28').toFixed(2);
```

### 5. Encryption and Security

Financial data demands the highest security standards.

**Core Skills:**
- AES-256-GCM encryption for data at rest
- TLS 1.3 for data in transit

- OWASP Top 10 awareness and mitigation
- Certificate pinning in Electron
- Secure IPC channel design (no arbitrary code execution)
- Key derivation functions (PBKDF2, Argon2) for local encryption keys
- SOC 2 compliance requirements and audit preparation

**Learning Resources:**
- OWASP Electron Security Checklist
- Supabase Row Level Security documentation
- "Bulletproof SSL and TLS" by Ivan Ristic

## Domain Skills (US Tax Code)

### 7. Federal Income Tax

The core domain knowledge for Taxonaut.

**Must-Know IRS Publications:**
- **Publication 535** -- Business Expenses (the bible for freelancer deductions)
- **Publication 587** -- Business Use of Your Home
- **Publication 463** -- Travel, Gift, and Car Expenses
- **Publication 560** -- Retirement Plans for Small Business
- **Publication 334** -- Tax Guide for Small Business
- **Publication 505** -- Tax Withholding and Estimated Tax

**Key Tax Forms:**
- **Schedule C** (Form 1040) -- Profit or Loss from Business (Sole Proprietorship)
- **Schedule SE** -- Self-Employment Tax
- **Form 1040-ES** -- Estimated Tax for Individuals
- **Form 8829** -- Expenses for Business Use of Your Home
- **Form 4562** -- Depreciation and Amortization (Section 179)
- **Form 2553** -- Election by a Small Business Corporation (S-Corp election)

**Critical Concepts:**
- Progressive tax brackets (2024: 10%, 12%, 22%, 24%, 32%, 35%, 37%)
- Standard deduction vs itemized deductions
- Self-employment tax (15.3%: 12.4% Social Security + 2.9% Medicare)
- Qualified Business Income (QBI) deduction (Section 199A) -- 20% of qualified income
- Above-the-line deductions (SEP IRA, self-employed health insurance, 1/2 SE tax)
- Alternative Minimum Tax (AMT) awareness
- Net Operating Loss (NOL) carryforward rules
- Wash sale rules for investment losses

### 8. Business Entity Types

Understanding entity types is critical for the Entity Analyzer feature.

**Entity Comparison Knowledge:**

| Entity | Self-Employment Tax | Legal Protection | Compliance Cost | Best For |
|--------|-------------------|-----------------|-----------------|----------|
| Sole Proprietor | Full (15.3% on all net income) | None | $0 | Income < $50K |
| Single-Member LLC | Full (15.3% on all net income) | Yes | $50-800/yr (state fees) | Income $50K-80K |
| S-Corporation | Only on "reasonable salary" | Yes | $2,000-5,000/yr | Income > $80K |
| C-Corporation | None (corp pays corp tax) | Yes | $3,000-8,000/yr | VC-funded, retained earnings |

**S-Corp Deep Dive:**
- "Reasonable salary" determination (IRS scrutinizes this)
- Payroll requirements (quarterly 941s, annual W-2)
- Shareholder distribution mechanics
- Form 2553 filing deadline (March 15 for calendar year, or 2 months 15 days into new entity)
- Late S-Corp election relief procedures
- State-level S-Corp recognition variations

### 9. Deduction Categories

Deep knowledge of what qualifies as a business deduction.

**Common Deduction Areas:**
- Home office (simplified method: $5/sq ft up to 300 sq ft; actual method: proportional expenses)
- Vehicle (standard mileage rate vs actual expenses -- cannot switch methods)
- Meals (50% deductible if business purpose documented)
- Travel (100% deductible if primarily business)
- Health insurance (100% deductible for self-employed, above-the-line)
- Retirement contributions (SEP IRA: 25% of net SE income up to $69,000)
- Software and subscriptions (100% deductible if business use)
- Professional development (courses, books, conferences)
- Equipment (Section 179 immediate expensing up to $1,220,000 in 2024)
- Depreciation (MACRS for assets not expensed under Section 179)
- Internet and phone (business percentage only)
- Professional services (legal, accounting, consulting)
- Business insurance (liability, E&O, health)
- Advertising and marketing

**Edge Cases to Know:**
- Mixed-use assets (laptop used 70% business, 30% personal)
- Hobby loss rules (IRS 3-of-5-year profit test)
- Meals that are NOT deductible (commuting, groceries)
- Home office for renters vs homeowners
- Clothing (almost never deductible unless uniform/costume)
- Pet expenses (almost never deductible, but there are narrow exceptions)

### 10. Quarterly Estimated Payments

**Critical Knowledge:**
- Four quarterly deadlines: April 15, June 15, September 15, January 15 (next year)
- Safe harbor: Pay 100% of prior year tax (110% if AGI > $150K) OR 90% of current year
- Penalty calculation: IRS charges ~8% annualized on underpayments
- Annualized income installment method for irregular income
- State estimated taxes (different schedules in some states)
- EFTPS (Electronic Federal Tax Payment System) for making payments
- First-year freelancers have no prior year liability (only current year safe harbor applies)

---

### 11. State Tax Variations

**Key State Differences:**
- No income tax states: AK, FL, NV, NH (dividends only), SD, TN (dividends only), TX, WA, WY
- California: highest top rate (13.3%), franchise tax minimum ($800 for LLCs)
- New York: state + city tax for NYC residents
- Nexus rules: physical presence vs economic nexus thresholds
- State-specific deductions and credits
- Reciprocity agreements (e.g., VA/DC/MD)
- Pass-through entity tax elections (SALT workaround)

## Design Skills

### 12. Financial Dashboard Design

**Core Skills:**
- Information hierarchy for financial data (most important numbers biggest and most prominent)
- Data-dense layouts that are not overwhelming
- Number formatting consistency (right-aligned, tabular figures, consistent decimals)
- Color coding for financial context (green for savings/income, red for expenses/overdue, amber for warnings)
- Sparklines and micro-charts for trend indicators
- Gauge visualizations for tax rate and progress indicators
- Responsive charts with proper axis labeling and tooltips

**Design Principles for Financial Apps:**
- Accuracy over aesthetics (numbers must be exactly right, never rounded in misleading ways)
- Consistency in number formatting across every screen
- Adequate whitespace to prevent misreading adjacent numbers
- Monospaced or tabular-lining fonts for all financial figures
- Currency symbols always present (never assume the user knows it is dollars)

---

### 13. Trust-Building UI

Users are sharing their most sensitive financial data. The UI must actively build trust.

**Trust Indicators:**
- Security badges visible on every screen (encryption status, connection security)
- "Last synced" timestamps showing data freshness
- Transparent data access disclosures ("Taxonaut can see: transactions, balances. Taxonaut CANNOT see: passwords, SSN")
- Clear privacy controls (what data is stored, how to delete it)
- Professional visual design (no gimmicks, no gamification -- this is money)
- Error messages that are honest and actionable
- Uptime and sync status indicators
- Third-party security certifications displayed (SOC 2 badge when achieved)

**Anti-Patterns to Avoid:**
- Dark patterns (tricky upsells, hidden charges)
- Gamification of financial data (no streaks, no badges, no confetti)
- Overpromising savings ("guaranteed" is a dangerous word in finance)
- Hiding disclaimers (they must be visible, not buried in fine print)

---

### 14. Data Visualization for Money

**Chart Types Used in Taxonaut:**
- Bar charts: Income vs expenses (monthly comparison)
- Line charts: Tax liability over time (trend)
- Donut charts: Expense category breakdown
- Gauge charts: Effective tax rate, progress toward deduction thresholds
- Stacked area charts: Income source breakdown
- Horizontal bar charts: Deduction categories
- Comparison tables: Entity type analysis

**Visualization Rules:**
- Y-axis always starts at zero for bar charts (never truncate to exaggerate)
- Dollar amounts on axes use K/M suffixes for readability ($12K, $1.2M)
- Color palette is colorblind-safe (test with Coblis or similar tools)
- Hover states show exact values (charts are for trends, hover is for precision)
- All charts have an accessible data table toggle for screen readers

## Business Skills

### 15. Fintech Compliance

**Regulatory Awareness:**
- Taxonaut does NOT need money transmitter licenses (it does not move money)
- Taxonaut does NOT need to register as a CPA firm (it provides suggestions, not advice)
- Taxonaut DOES need clear disclaimers on every recommendation
- Taxonaut SHOULD pursue SOC 2 Type II certification for customer trust
- State-specific consumer protection laws may apply
- FTC guidelines on endorsements and testimonials (if showing user savings testimonials)
- GLBA (Gramm-Leach-Bliley Act) may apply to financial data handling
- State privacy laws (CCPA in California, others emerging)

**Disclaimer Requirements:**
Every strategy recommendation must include:
```
This is a tax education suggestion, not tax advice. Consult a qualified
tax professional before making tax decisions. Taxonaut is not a CPA firm
or registered tax advisor. Tax laws change frequently -- verify current
applicability with a professional.
```

## Unique Skills (Hard to Find)

### 19. AI + Tax Code Intersection

The rarest and most valuable skill: someone who understands both AI engineering AND the US tax code well enough to design prompts, validate outputs, and build guard rails.

**What This Looks Like:**
- Designing GPT-4o prompts that reference specific IRS publications
- Building validation layers that catch AI hallucinations about tax rules
- Creating test suites with known tax scenarios and expected outcomes
- Understanding where AI excels (categorization, natural language explanation) vs where deterministic rules are better (bracket calculations, contribution limits)
- Knowing when the AI is wrong (and it WILL be wrong sometimes -- especially on edge cases)

**Guard Rails Required:**
- Every AI-generated strategy must pass through a rules engine validation
- Dollar amounts in recommendations must be independently verified by deterministic calculation
- IRS code references must be validated against a known-good database
- Recommendations must be flagged with confidence levels
- A human CPA advisor reviews new strategy types before they go live

---

### 20. Electron + Fintech Security

Another rare combination: someone who understands Electron's security model AND financial data protection requirements.

**Specific Concerns:**
- Electron's Chromium renderer can be exploited if misconfigured
- nodeIntegration must ALWAYS be false in renderer
- contextIsolation must ALWAYS be true
- preload scripts must use contextBridge (never expose full Node.js API)
- Local SQLite databases must be encrypted (not just the file system)
- OS keychain must be used for ALL secrets (Plaid tokens, encryption keys)
- Auto-updater must verify code signatures to prevent supply chain attacks
- CSP headers must block inline scripts and unauthorized external resources
